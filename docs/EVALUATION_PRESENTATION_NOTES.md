# Evaluation Presentation Notes

## How To Describe The Interview Module

Use this wording during evaluation:

`The project conducts a real-time AI interview session, not a static mock form. The candidate grants camera and microphone access, the system generates role-specific questions, the candidate answers in real time, and each answer is analyzed for transcript quality, verbal communication, and non-verbal behavioral indicators.`

## Important Clarification

Do not describe the module as:

- only a demo interview
- only a mock practice screen
- a human-to-human live call

Describe it as:

- a real-time AI-guided interview session
- an automated interview workflow
- a live candidate response capture and analysis system

## What To Emphasize

1. Camera and microphone are accessed live in the browser
2. Questions are dynamically generated based on role and interview type
3. Each answer is recorded and analyzed during the session
4. Transcript, sentiment, clarity, confidence, and non-verbal indicators are included
5. Reports are persisted in MongoDB
6. Candidate history and recruiter comparison are available after the session

## What To Avoid Overclaiming

Do not say:

- the system can detect true emotions with certainty
- the system performs lie detection
- the system replaces final human hiring decisions

Say instead:

- it provides behavioral indicators
- it supports recruiter decision-making
- it improves consistency in early-stage evaluation

## Best Demo Sequence

1. Show job posting
2. Show resume upload
3. Show ATS report
4. Start real-time AI interview
5. Record one answer live
6. Show structured report
7. Show saved history
8. Show recruiter comparison
